What It Does

Brief Overview

This tool manages a live equity and mutual fund portfolio on the Nepal Stock Exchange (NEPSE). It processes 264+ transactions — 177 equity trades, 36 mutual fund purchases, and 51 SIP installments across 32 scrips — inside a single parameter-driven Excel workbook. One change to a commission slab, tax rate, or NAV cascades through ~2,100 dependent calculations automatically, eliminating ~85% of per-trade manual rework.

Tools Used

Stack

  • Microsoft Excel
  • Power Query
  • Pivot Tables
  • FIFO Engine
  • SUMIF / Dynamic Arrays
  • Web Connections

Why It Matters

Business Value

The project demonstrates enterprise-grade Excel architecture applied to a real financial problem: automated cost-basis tracking, live data integration, and audit-ready reconciliation — the same logic that underpins inventory management, supply-chain costing, and sales reporting systems. The parameter-driven schema is reusable across domains with fewer than 20% structural changes.

Key Features

What was built and the problems it solved.

Formula Architecture

Parameter-Driven Schema

Engineered a fully formula-driven workbook where a single parameter change — commission slab, tax rate, or NAV — cascades through ~2,100 dependent calculations. This eliminated ~85% of per-trade manual rework and removed the risk of inconsistent updates across tabs.

Live Data

Real-Time NEPSE Price Feed

Integrated live NEPSE price feeds via Power Query web connections that refresh 330 tickers in under 30 seconds. This replaced a ~25-minute daily manual price-update routine and ensures valuations are always current without any manual intervention.

Cost Tracking

FIFO-Weighted Cost-of-Sales Engine

Built a FIFO engine using running SUMIF with dynamic sold-stock weightage across all 264 transactions. It automates realised-gain, WACC, and per-lot balance-cost calculations — removing the most error-prone manual step (cost-basis tracking) and cutting per-trade calculation time from ~3 minutes to instant.

Reconciliation

Dual-Layer Reconciliation

Designed a two-track reconciliation mechanism: book holdings vs. broker DP balances across 32 scrips, and sectoral cost vs. total investment across 14 sectors. This cut reconciliation time from ~90 minutes to under 5 minutes per cycle and surfaces discrepancies immediately.

Dashboards

Decision-Ready Reporting

Built pivot-table dashboards covering sectoral exposure, realised P&L by ticker, and rule-based Buy/Sell/Hold signals banded against per-scrip price thresholds. Consolidated 10 raw data tabs into a single one-screen view for fast, decision-oriented reading.

Transferability

Reusable Design Logic

Architected with lookup tables for fees, tax bands, and strategy thresholds — the same schema is reusable for inventory reconciliation, supply-chain costing, or sales tracking with fewer than 20% structural changes. Built to demonstrate scalable, transferable design thinking beyond the finance domain.
